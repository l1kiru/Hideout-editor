## Что нужно установить

- Python 3.10 или новее
- Node.js 22 (LTS) или новее — в CI закреплён Node 22
- npm 10 или новее — ставится вместе с Node.js 22
- Git, только если вы хотите обновлять проект через `git pull`

## Как запустить

Откройте папку проекта после скачивания архива или клонирования репозитория и выполните:

```bash
python dev.py
```

На macOS или Linux используйте эту команду, если `python` не найден или указывает на Python 2:

```bash
python3 dev.py
```

На Windows можно просто дважды нажать `Start.bat`.

Первый запуск может занять время. Скрипт сам подготовит проект:

1. создаст `.venv` для Python-пакетов;
2. установит зависимости бэкенда из `requirements-api.txt`;
3. установит зависимости фронтенда в `frontend/node_modules`;
4. запустит бэкенд и фронтенд.

Когда в терминале появится сообщение, что всё запущено, откройте:

- приложение: http://localhost:5173
- документация API: http://127.0.0.1:8000/api/docs
- проверка бэкенда: http://127.0.0.1:8000/api/health

Если на вашей системе команда `python` уже указывает на Python 3, можно
использовать корневую npm-обёртку:

```bash
npm start
```

Если на машине есть только `python3`, лучше запускать `python3 dev.py`
напрямую, потому что корневой npm-скрипт всё равно вызывает `python`.

Чтобы остановить приложение, вернитесь в терминал и нажмите `Ctrl+C`.

## Обновление

Если проект был скачан через Git:

```bash
git pull
python dev.py
```

Если на системе есть только `python3`, после `git pull` используйте
`python3 dev.py`.

На Windows можно использовать `UpdateAndStart.bat`: он сначала делает `git pull`, а потом запускает приложение.

Если после обновления появились странные ошибки или поменялись зависимости, пересоберите локальное окружение:

```bash
python dev.py --reinstall
```

Если на системе есть только `python3`, используйте
`python3 dev.py --reinstall`.

Ваши локальные данные лежат в `hideout_settings/`, `hideout_scenes/`, `input/images/` и `logs/`. Эти папки не добавляются в Git и не попадают в релизный архив. Исключение — папка `input/hideout/`: в ней лежат подготовленные образцы `.hideout`-карт, которые используются как базовые (см. [Страницу разметки](#страница-разметки)).

## Полезные параметры запуска

```bash
python dev.py --skip-install  # быстрый запуск без переустановки пакетов
python dev.py --reinstall     # удалить .venv и frontend/node_modules, потом поставить заново
python dev.py --dev           # автоперезапуск бэкенда при разработке
python dev.py --no-browser    # не открывать браузер автоматически
```

Порты можно поменять через переменные окружения:

| Переменная                     | По умолчанию | Что меняет     |
| ------------------------------ | ------------ | -------------- |
| `HIDEOUT_EDITOR_BACKEND_HOST`  | `127.0.0.1`  | хост бэкенда   |
| `HIDEOUT_EDITOR_BACKEND_PORT`  | `8000`       | порт бэкенда   |
| `HIDEOUT_EDITOR_FRONTEND_PORT` | `5173`       | порт фронтенда |

## Страница разметки

Страница разметки границ (первый скриншот выше) превращает дамп тайника
(`.hideout`) в замкнутый полигон, описывающий игровую зону. Откройте
приложение и перейдите на вкладку **Разметка**.

**Загрузка исходного файла.** Два варианта:

1. **Загрузить .hideout** — системный диалог выбора файла. Используйте
   для собственных экспортов из игры.
2. **Примеры карт** — выпадающий список рядом с кнопкой загрузки. Любой
   образец из папки [`input/hideout/`](input/hideout/) проходит тот же
   парсер, что и обычная загрузка. В этой папке лежат подготовленные
   мной базовые карты — берите их как стартовую точку или эталон.

**Построение контура.** После загрузки:

1. Страница автоматически берёт самый частый тип doodad'а как маркер
   границы (тот тип, которым вы вручную выкладывали контур зоны в игре).
   Переопределить можно в разделе **Маркер контура зоны → Выбор объекта
   разметки** или вписав имя doodad'а вручную.
2. Превью полигона справа показывает точки маркера (зелёные) и
   остальную карту (бледные). Вершина выбирается кликом или рамкой,
   колесом мыши над полигоном выбранная вершина двигается по кругу
   порядка.
3. Кнопки в разделе **Порядок вершин** переставляют вершины по одной,
   разворачивают кольцо или применяют автоматическую эвристику
   («ближайший сосед», «угол от центра»).
4. Контур должен замыкаться без самопересечений — красный пунктирный
   отрезок означает пересечение, его нужно исправить до сохранения.

**Сохранение.** Введите имя карты и нажмите **Записать в базу**.
Отметьте _Создать как базовую карту проекта_, если карта должна
появиться в списке базовых в редакторе. После этого можно перейти на
вкладку редактора, разместить декорации поверх и экспортировать готовый
`.hideout`.

## Структура проекта

- `backend/` — FastAPI-приложение, хранение карт, import/export endpoints.
- `frontend/` — интерфейс на React/Vite.
- `hideout_core/` — общая Python-логика.
- `backend/seed_data/` — каталог данных, нужный бэкенду.
- `shared/` — generated-артефакты контракта сцены и каталог ассетов редактора.
- `docs/` — установка, обновление и решение частых проблем.
- `input/hideout/` — подготовленные базовые карты `.hideout`, доступные
  на странице разметки как **Примеры карт**.
- `scripts/` — служебные скрипты для сопровождения проекта.
- `tests/` — тесты бэкенда.
